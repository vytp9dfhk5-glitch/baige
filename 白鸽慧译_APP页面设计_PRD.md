# 白鸽慧译 APP 页面设计需求说明书

## Problem Statement

白鸽慧译一期需要把“会议录音 / 音频记录 / AI 纪要 / 摘要图 / 历史回看 / 设备设置”收敛成一个清晰的 APP 闭环。当前页面设计已经从早期展示型原型演进为更接近真实使用路径的移动端产品：用户进入首页后应立即开始录音；录音结束后选择模板并生成纪要；用户可在历史中搜索、分页查看过往会议；进入详情后可听音频、看转写、看 AI 纪要，并在 AI 纪要中切换模板重新生成报告。

用户当前关注的问题是：页面信息需要更聚焦，录音、历史、详情、设置的职责边界需要明确；图文纪要不能只是一张图片，还必须包含缩略摘要图、可放大/修改操作和文字版会议纪要描述；AI 纪要模板切换也不能直接切换，应先弹窗确认，且只有选择不同模板时才进入重新生成状态。

## Solution

白鸽慧译 APP 一期页面由三个底部主模块组成：首页、历史、设置。会议详情页作为从首页录音结束或历史记录点击进入的二级页面，不在底部导航中单独暴露。

首页以录音为唯一核心任务。顶部展示放大的产品名称“白鸽慧译”和硬件设备状态，状态使用绿色灯表示正常，红色灯表示异常。页面中部为录音按钮和录音计时。初始状态只展示开始录音；点击开始后显示暂停和结束按钮；暂停时录音按钮图标应变为暂停状态；点击结束后弹出模板选择弹窗，默认选中标准会议纪要。用户确认后进入会议详情页，详情默认展示 AI 纪要。

历史模块仅展示历史会议搜索和会议列表。用户可按会议标题或日期进行模糊搜索。默认展示近 10 条记录，点击“查看更多”再展开 10 条，以此类推。点击任一历史会议记录后进入会议详情页。

会议详情页参考当前上传的纪要页面样式：顶部包含返回、会议标题和更多操作；下方为音频播放器，播放器按钮使用一致的图标风格；详情页仅保留“转写”和“AI纪要”两个标签。AI 纪要默认展示当前模板报告，模板入口显示当前模板名称。点击模板入口时弹出模板选择弹窗，默认选中当前模板。若用户选择相同模板并确认，不重新生成；若选择不同模板并确认，先展示“正在生成纪要...”加载动画，再按新模板展示报告。

图文纪要由两部分组成：一张缩放嵌入在纪要正文中的摘要长图，以及其下方的文字版会议纪要描述。摘要图在详情中以缩略比例展示，允许点击放大查看，并提供修改入口。文字版会议纪要需要包含会议主题、时间、参与人，以及结构化正文内容。

设置模块仅保留硬件连接状态和设备名称设置。设备名称区域不展示额外说明文案。

## User Stories

1. As a 会议参与人, I want 首页进入后直接看到开始录音按钮, so that 我可以快速开始会议记录。
2. As a 会议参与人, I want 首页顶部看到“白鸽慧译”产品名称, so that 我明确当前正在使用的产品。
3. As a 会议参与人, I want 首页顶部看到设备连接状态, so that 我在录音前确认硬件是否可用。
4. As a 会议参与人, I want 设备正常时看到绿色状态灯, so that 我知道可以开始录音。
5. As a 会议参与人, I want 设备异常时看到红色状态灯, so that 我能及时处理硬件问题。
6. As a 会议参与人, I want 初始首页只展示开始录音入口, so that 页面不被暂停、结束等未到达状态干扰。
7. As a 会议参与人, I want 点击开始录音后看到暂停和结束按钮, so that 我可以控制录音过程。
8. As a 会议参与人, I want 录音时看到声纹播放特效, so that 我知道系统正在采集声音。
9. As a 会议参与人, I want 录音时看到录音时间, so that 我能掌握会议记录时长。
10. As a 会议参与人, I want 点击暂停后录音图标变成暂停图案, so that 我能明确当前录音状态。
11. As a 会议参与人, I want 点击暂停后可继续录音, so that 临时中断不会结束会议记录。
12. As a 会议参与人, I want 点击结束后弹出模板选择, so that 我能决定本次会议生成哪种纪要。
13. As a 会议参与人, I want 录音结束模板弹窗默认选中标准会议纪要, so that 默认路径符合多数正式会议场景。
14. As a 会议参与人, I want 可选择标准会议纪要, so that 我能获得摘要、结论、待办和风险点。
15. As a 会议参与人, I want 可选择头脑风暴模板, so that 创意讨论可以保留灵感点和优选方向。
16. As a 会议参与人, I want 可选择图文纪要模板, so that 我能得到适合群内同步的摘要图和文字说明。
17. As a 会议参与人, I want 确认模板后进入会议详情页, so that 我能马上查看生成结果。
18. As a 会议参与人, I want 进入详情页默认展示 AI 纪要, so that 我优先看到整理后的会议结果。
19. As a 会议参与人, I want 详情页可以切换到转写, so that 我能追溯原始会议内容。
20. As a 会议参与人, I want 详情页顶部看到会议标题, so that 我明确正在查看哪一场会议。
21. As a 会议参与人, I want 详情页顶部有返回入口, so that 我可以回到首页或历史列表。
22. As a 会议参与人, I want 详情页右上角仅保留更多操作, so that 页面保持简洁。
23. As a 会议参与人, I want 详情页音频播放器展示开始时间、总时长和播放进度, so that 我能定位音频位置。
24. As a 会议参与人, I want 播放按钮使用图标而不是文字, so that 操作更符合移动端播放器习惯。
25. As a 会议参与人, I want 播放按钮与左右控制按钮图标风格一致, so that 播放器视觉统一。
26. As a 会议参与人, I want AI 纪要页面只保留“转写”和“AI纪要”两个标签, so that 信息结构更清晰。
27. As a 会议参与人, I want AI 纪要模板入口显示当前模板名称, so that 我知道当前报告的生成方式。
28. As a 会议参与人, I want 点击 AI 纪要模板入口后弹窗选择模板, so that 模板切换前有明确确认动作。
29. As a 会议参与人, I want 模板选择弹窗默认选中当前模板, so that 我不会误以为当前模板改变了。
30. As a 会议参与人, I want 选择相同模板后确认不重新生成, so that 系统避免无意义等待。
31. As a 会议参与人, I want 选择不同模板后确认才重新生成, so that 模板变更是主动行为。
32. As a 会议参与人, I want 重新生成时看到生成中动画, so that 我知道系统正在处理。
33. As a 会议参与人, I want 重新生成完成后看到新模板报告, so that 我能对比不同模板的输出价值。
34. As a 会议参与人, I want 图文纪要包含缩略摘要长图, so that 我能快速浏览会议重点图。
35. As a 会议参与人, I want 图文纪要的摘要图嵌入在纪要正文中, so that 图和文字在同一阅读流里。
36. As a 会议参与人, I want 摘要图支持放大查看, so that 我能看清长图中的细节。
37. As a 会议参与人, I want 摘要图支持修改入口, so that 我可以调整图文纪要内容。
38. As a 会议参与人, I want 图文纪要下方还有文字版会议纪要, so that 图片不能替代完整文字内容。
39. As a 会议参与人, I want 图文纪要文字版包含会议主题、时间和参与人, so that 纪要信息完整。
40. As a 会议参与人, I want 图文纪要文字版包含结构化章节, so that 我可以快速理解会议背景、目标和行动方向。
41. As a 会议参与人, I want 标准会议纪要展示会议摘要、会议结论和待办事项, so that 正式会议结果可直接用于同步。
42. As a 会议参与人, I want 头脑风暴纪要展示讨论主题、灵感点和优选方向, so that 发散讨论可以沉淀为可推进事项。
43. As a 会议参与人, I want 转写内容按发言人展示, so that 我能区分不同发言片段。
44. As a 会议参与人, I want 转写内容保留时间点, so that 我能和音频播放位置对应。
45. As a 会议组织者, I want 历史模块支持搜索会议标题, so that 我能快速找到某次会议。
46. As a 会议组织者, I want 历史模块支持搜索会议日期, so that 我能按时间查找会议。
47. As a 会议组织者, I want 搜索按钮和搜索输入框在同一行, so that 搜索操作紧凑清晰。
48. As a 会议组织者, I want 历史列表默认展示 10 条, so that 页面不会一次展示过多记录。
49. As a 会议组织者, I want 点击查看更多后再展示 10 条, so that 我可以按需浏览更多历史。
50. As a 会议组织者, I want 搜索无结果时看到空状态, so that 我知道当前关键词没有匹配会议。
51. As a 会议组织者, I want 点击历史记录后进入详情页, so that 列表页保持简洁。
52. As a 会议组织者, I want 历史记录展示标题、创建人、时间、模板和时长, so that 我在进入详情前能判断记录是否正确。
53. As a 会议组织者, I want 从历史进入详情后默认展示 AI 纪要, so that 我快速查看会议整理结果。
54. As a 设备使用者, I want 设置页查看硬件连接状态, so that 我能确认设备是否可用于录音。
55. As a 设备使用者, I want 设置页展示设备电量和信号状态, so that 我能提前处理设备风险。
56. As a 设备使用者, I want 设置页修改设备名称, so that 我能区分不同硬件。
57. As a 设备使用者, I want 设备名称区域不展示额外说明文案, so that 设置页更加简洁。

## Implementation Decisions

- APP 采用底部三模块导航：首页、历史、设置。
- 会议详情页作为二级页面，由首页录音完成后进入，或由历史记录点击进入。
- 首页顶部只展示产品名称和设备状态，不展示“首页”字样。
- 首页设备状态使用状态灯表达，绿色表示正常，红色表示异常。
- 首页初始状态只展示开始录音主按钮，不展示暂停和结束按钮。
- 开始录音后展示暂停和结束按钮，同时启动声纹播放动效和录音计时展示。
- 暂停录音时，主录音按钮图标切换为暂停图案。
- 结束录音后弹出模板选择弹窗，默认选中标准会议纪要。
- 录音结束模板弹窗包含标准会议纪要、头脑风暴、图文纪要三个模板。
- 用户确认录音结束模板后进入会议详情页，默认激活 AI 纪要标签。
- 历史页不展示英文副标题、顶部统计或快捷卡片，仅保留中文标题、搜索和列表。
- 历史搜索支持按标题和日期模糊匹配。
- 历史列表默认展示 10 条，查看更多每次增加 10 条。
- 历史列表点击单条记录后进入会议详情页。
- 会议详情页顶部只保留返回和更多操作。
- 会议详情页不展示搜索图标和分享图标。
- 会议详情页仅保留“转写”和“AI纪要”标签，不展示章节和分析。
- 详情页播放器中间按钮使用播放三角图标，按钮风格与左右控制按钮一致。
- AI 纪要模板入口展示当前模板名称。
- AI 纪要模板入口点击后打开模板选择弹窗。
- AI 纪要模板弹窗默认选中当前模板。
- AI 纪要模板弹窗中选择相同模板并确认时，不触发重新生成。
- AI 纪要模板弹窗中选择不同模板并确认时，显示“正在生成纪要...”加载动画，再展示新模板报告。
- 图文纪要包含缩略嵌入的摘要长图，长图不替代文字纪要。
- 图文纪要的摘要图需要提供放大查看入口。
- 图文纪要的摘要图需要提供修改入口。
- 图文纪要摘要图放大后以覆盖层展示，可关闭回到详情。
- 图文纪要文字版展示会议主题、时间、参与人及结构化正文。
- 设置模块只包含硬件连接状态和设备名称。
- 设置模块删除录音提醒、设备介绍卡片和设备名称辅助说明。
- 当前原型中的关键状态机可作为实现参考：

```text
home.initial -> start_recording -> home.recording
home.recording -> pause -> home.paused
home.paused -> resume -> home.recording
home.recording/home.paused -> finish -> template_modal
template_modal.confirm -> detail.summary
history.search -> filtered_history_list
history.item_click -> detail.summary
detail.summary.template_click -> summary_template_modal
summary_template_modal.confirm_same -> detail.summary.current
summary_template_modal.confirm_different -> detail.summary.generating -> detail.summary.new_template
```

## Testing Decisions

- 测试应以用户可见行为为主，不依赖内部 DOM 结构命名或具体 CSS 实现。
- 首页录音流程需要测试：初始只显示开始录音；点击开始后显示暂停和结束；点击暂停后图标状态变化；点击结束后弹出模板选择。
- 录音结束模板选择需要测试：默认选中标准会议纪要；选择不同模板后确认进入详情页；详情页默认展示 AI 纪要。
- 历史模块需要测试：标题搜索、日期搜索、无结果空状态、默认 10 条展示、查看更多追加 10 条、点击记录进入详情。
- 会议详情需要测试：默认展示 AI 纪要；可切换转写；右上角不展示搜索和分享；不展示章节和分析。
- 播放器需要测试：中间播放按钮是图标按钮，且与左右控制按钮视觉风格一致。
- AI 纪要模板切换需要测试：点击当前模板打开弹窗；弹窗默认选中当前模板；确认相同模板不展示生成中；确认不同模板展示生成中并切换内容。
- 图文纪要需要测试：缩略摘要图可见；存在放大入口；存在修改入口；放大层可关闭；缩略图下方展示文字版纪要内容。
- 设置页需要测试：仅展示硬件连接状态和设备名称；不展示录音提醒、设备介绍卡片、设备名称辅助说明。
- 建议使用端到端测试覆盖主流程：开始录音 -> 暂停 -> 结束 -> 选择模板 -> 详情 AI 纪要 -> 切换模板 -> 查看生成中 -> 查看新报告。
- 建议使用移动端视口截图回归，重点覆盖首页、历史、详情图文纪要、模板弹窗和设置页。

## Out of Scope

- 不包含真实录音能力实现。
- 不包含真实音频播放、音频进度控制和倍速播放实现。
- 不包含真实蓝牙硬件连接协议。
- 不包含真实 AI 服务接口、转写接口和图片生成接口。
- 不包含用户账号、权限、分享链接和协同编辑。
- 不包含 Word/PDF 导出。
- 不包含历史标签、归档和高级筛选。
- 不包含自定义模板编辑器。
- 不包含声纹识别、发言人自动命名和热词库管理。
- 不包含真实摘要图图片渲染服务，仅定义前端展示和交互需求。

## Further Notes

- 当前需求基于最新 APP 页面原型整理，原型文件为当前项目中的静态 HTML/CSS 实现。
- 设计风格参考用户提供的移动端截图：浅色背景、圆角播放器、简洁顶部栏、胶囊模板入口、纪要正文与摘要图组合展示。
- 图文纪要需要明确区分“摘要图”和“文字版纪要”：摘要图用于快速分享和视觉浏览，文字版纪要用于完整阅读和追溯。
