# 白鸽在线智能电子名片

一套单手机屏幕居中的电子名片应用原型，包含手机号验证码登录、自定义名片、PDF公司介绍长图、微信联系人选择、访客轨迹和角色管理。当前版本使用 React + Vite，可直接用于产品评审、交互演示，并可作为后续微信小程序开发的前端设计基线。

## 在线地址

- 管理端登录：<https://vytp9dfhk5-glitch.github.io/baige/ai-card/>
- 涂锦波访客名片：<https://vytp9dfhk5-glitch.github.io/baige/ai-card/?view=visitor>

访客链接是独立的只读窗口，打开后直接展示涂锦波的个人信息和管理员设置的公司介绍长图，不显示编辑、访客分析或“我的”等管理功能。

## 当前产品流程

1. 用户进入应用后首先看到手机号加验证码登录页。
2. 输入 11 位手机号、6位验证码并同意用户协议、隐私政策后登录。
3. 员工手机号与员工清单匹配后自动带入姓名、邮箱、登录手机号和默认职位；非员工清单手机号进入经纪人角色，不带预设信息。
4. 员工和经纪人填写相同的个人字段，系统与管理员维护的企业配置合并生成完整名片。
5. 员工不修改预设职位时，其他个人信息保存后无需审核；经纪人首次生成名片需管理员审核并设置有效期。
6. 员工和经纪人登录后包含“名片”“访客”“我的”三个模块；管理员额外包含“管理”模块。
7. 可用名片点击右上角分享按钮后，直接弹出微信联系人选择原型。

不同手机号使用独立的本地存储键，拥有各自独立的名片数据。

## 角色与权限

系统包含管理员、员工和经纪人三种角色：

- 管理员通过“企业配置”维护公司名称、公司地址、PDF公司介绍长图、企业宣传视频、视频封面和企业资讯；企业资讯可拖拽排序，配置需预览确认后才发布。
- 管理员可以添加成员、调整管理员/员工/经纪人角色，更新员工登录手机号，以及启用或停用账号。
- 员工与经纪人可填写相同的个人字段；员工有清单预设信息，经纪人没有预设信息。
- 员工名片展示手机号可以自行修改，不会改变员工清单中的登录手机号；登录手机号变更需联系管理员后台更新。
- 员工未修改预设职位时，修改其他个人信息无需审批；员工首次名片默认有效期为一年。
- 经纪人首次生成名片必须经过管理员审核，审核时由管理员设置有效期，未通过或过期均不可使用。
- 名片在到期前最后 7 天至过期后可提交续期申请；管理员通过时设置新有效期，拒绝后名片立即进入已禁用状态。
- 个人名片由“个人信息 + 企业配置”自动合并，个人编辑页不显示企业字段。
- 默认主管理员手机号为 `18059880224`；员工清单外手机号在原型中自动进入经纪人流程。

当前原型使用浏览器本地存储演示角色和权限。正式上线时需要由后端鉴权接口签发角色权限，前端不能作为最终安全边界。

管理员审批中心默认只展示按提交时间倒序排列的最新 5 条待审核申请。点击列表底部“更多”进入独立的全部申请页，可按申请类型和状态筛选待审核、已通过和已拒绝记录。拒绝申请时必须填写拒绝原因。

## 默认公司介绍

附件《【企业简介】白鸽在线（2026版）.pdf》共15页，原PDF已保存为 `public/company-introduction-2026.pdf`，并按每页1600×900渲染、合成为1600×13724的WebP长图 `public/company-introduction-2026.webp`。管理员配置页可查看或上传PDF并预览长图；员工和访客名片直接展示该长图。

## 自定义名片字段

员工个人信息仅以下两个字段必填：

- 姓名
- 职位

其余个人字段均为选填：

- 6个默认头像或头像上传
- 个人电话
- 个人邮箱
- 个人微信
- 个人简介
- 多个个人地址，支持关键词模糊搜索

公司名称、公司地址、PDF公司介绍、企业宣传视频和企业资讯由管理员在“企业配置”中统一维护。

应用会根据用户实际填写的内容生成名片。未填写的联系方式、个人简介、视频和资讯不会产生空模块。

头像上传后会在浏览器中自动压缩，再保存到当前手机号对应的本地名片数据中。

## 名片模块

- 首次登录只显示完善个人信息的空状态，不再展示“三步完成”。
- 生成后显示姓名、职位和公司名称。
- 根据填写情况显示头像、电话、邮箱、微信、地址和个人简介。
- 展示管理员配置的PDF公司介绍长图、企业宣传视频和企业资讯。
- 支持继续编辑名片。
- 右上角支持快速分享名片并弹出微信联系人选择原型。
- 名片页右上角提供独立消息通知入口，用于查看职位变更、创建名片、续期审批结果以及到期前 7 天、3 天、1 天和已过期提醒。
- 名片头像右上方支持“添加至通讯录”，自动生成姓名、公司名称、职位和电话等联系人资料。
- 名片首页明确显示“正常 / 待审核 / 已过期 / 已禁用”当前状态；到期前最后 7 天显示续期入口。
- 不提供打电话按钮，不展示核心业务“统一配置”或90亿保单等额外统计模块。
- 不再在名片页重复展示访客详情；访客数据统一进入独立“访客”模块。

## 访客模块

- 今日访问人数
- 近七天访问趋势
- 浏览总时长
- 重点联系人数量
- 单个访客访问次数
- 最关注内容
- 浏览轨迹及时间节点
- 关注度判断与沟通建议

当前访客列表为演示数据。正式版需要后端事件采集和微信用户授权。

## 我的模块

- 显示脱敏登录手机号
- 显示当前名片状态
- 退出登录

个人信息编辑统一从“名片”模块进入，“我的”不再提供重复编辑入口或“隐私与数据”入口。

## 消息通知与续期

- 消息通知属于当前名片持有人的个人消息，与管理员审批中心完全分开。
- 审批通过消息显示生效结果和新有效期；审批拒绝消息显示管理员填写的拒绝原因。
- 到期提醒按当前剩余有效期显示最后 7 天、3 天、1 天或已过期提示。
- 到期前最后 7 天至已经过期期间可提交一条待审核续期申请；重复提交按钮会锁定。
- 续期通过后名片按管理员设置的新有效期恢复正常；续期拒绝后名片进入已禁用状态，可重新提交续期申请。

## 企业配置发布

- 企业资讯支持通过拖拽调整展示顺序。
- 宣传视频支持 MP4、MOV、M4V、WebM 等常用格式，并可单独上传 16:9 视频封面。
- 管理员点击“预览效果”后，以示例个人信息拼接企业配置草稿，查看完整名片效果。
- 只有在预览页点击“确认发布”，配置才会应用到全部可用名片。

## 微信分享说明

网页原型点击右上角分享按钮后，会在手机软件画面内部直接弹出微信最近联系人选择界面。选择联系人后复制专属名片链接并显示发送成功反馈。

正式微信小程序应使用微信原生分享能力：

- 页面分享配置：`onShareAppMessage`
- 分享按钮：`open-type="share"`
- 分享卡片中携带名片所有者 ID 或短链接参数

网页无法读取真实微信联系人，因此当前列表属于交互原型；正式小程序使用微信原生分享面板，由用户选择真实联系人。

## 本地运行

需要 Node.js 18 或更高版本：

```bash
npm install
npm run dev
```

浏览器打开：

```text
http://localhost:5173/
```

生产构建：

```bash
npm run build
```

构建结果位于 `dist/`。

源码保存在 `baige` 仓库的 `ai-card-source` 分支。向该分支推送后，GitHub Actions 会重新构建，并只更新 `gh-pages` 分支中的 `ai-card/` 子目录，不会覆盖原“白鸽慧译”主页。

## 正式上线需要补充

1. 微信手机号授权登录或短信验证码登录。
2. 后端用户、名片、内容和访客数据 API。
3. 图片、视频封面和资料文件的对象存储。
4. 微信 OpenID/UnionID 与内部账号绑定。
5. 名片唯一公开地址、名片 ID 和分享短链接。
6. 页面打开、模块曝光、内容点击、联系方式点击、分享和退出等事件采集。
7. 使用页面可见性和心跳机制计算真实浏览时长。
8. 访客授权、匿名化、数据留存和删除机制。
9. 名片内容审核、敏感信息检测和后台管理权限。
