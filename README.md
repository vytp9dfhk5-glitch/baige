# 白鸽在线智能电子名片

一套手机端优先的电子名片应用原型，包含手机号快速登录、自定义名片、动态内容生成、微信分享、访客轨迹和个人账号管理。当前版本使用 React + Vite，可直接用于产品评审、交互演示，并可作为后续微信小程序开发的前端设计基线。

## 在线地址

- 管理端登录：<https://vytp9dfhk5-glitch.github.io/baige/ai-card/>
- 涂锦波访客名片：<https://vytp9dfhk5-glitch.github.io/baige/ai-card/?view=visitor>

访客链接是独立的只读窗口，打开后直接展示涂锦波的个人简介、公开联系方式、公司核心业务和企业数据，不显示编辑、分享、访客分析或“我的”等管理功能。

## 当前产品流程

1. 用户进入应用后首先看到手机号快速登录页。
2. 输入 11 位手机号并同意用户协议、隐私政策后登录。
3. 首次登录没有默认个人名片，进入“完善个人信息”空状态。
4. 用户填写个人资料，系统与管理员配置的企业介绍合并生成完整名片。
5. 员工登录后包含“名片”“访客”“我的”三个模块；管理员额外包含“管理”模块。
6. 生成名片后，可以通过微信分享面板发送给联系人。

不同手机号使用独立的本地存储键，拥有各自独立的名片数据。

## 角色与权限

系统包含管理员和员工两种角色：

- 管理员可以配置企业介绍，包括公司名称、公司地址、企业简介、公司资质、核心业务、企业宣传视频和企业资讯。
- 管理员可以添加员工、调整管理员/员工角色，以及启用或停用账号。
- 员工只可以配置自己的头像、姓名、职位、个人电话、个人邮箱、个人微信、个人简介和多个个人地址。
- 员工名片由“个人信息 + 管理员企业介绍”自动合并，企业内容在个人编辑页中锁定。
- 默认主管理员手机号为 `18059880224`；其他未提前配置的手机号首次登录时自动登记为员工。

当前原型使用浏览器本地存储演示角色和权限。正式上线时需要由后端鉴权接口签发角色权限，前端不能作为最终安全边界。

## 自定义名片字段

员工个人信息仅以下两个字段必填：

- 姓名
- 职位

其余个人字段均为选填：

- 头像上传
- 个人电话
- 个人邮箱
- 个人微信
- 个人简介
- 多个个人地址

公司名称、公司地址、企业简介、公司资质、公司核心业务、企业宣传视频和企业资讯由管理员统一配置。

应用会根据用户实际填写的内容生成名片。未填写的联系方式、个人简介、核心业务、视频和资讯不会产生空模块。

头像上传后会在浏览器中自动压缩，再保存到当前手机号对应的本地名片数据中。

## 名片模块

- 首次登录显示空状态和创建引导。
- 生成后显示姓名、职位和公司名称。
- 根据填写情况显示头像、电话、邮箱、微信、地址和个人简介。
- 展示管理员配置的企业简介、公司资质、核心业务、企业宣传视频和企业资讯。
- 支持继续编辑名片。
- 支持分享名片。
- 分享链接打开后支持一键复制姓名、公司名称、职位和电话。
- 不再在名片页重复展示访客详情；访客数据统一进入独立“访客”模块。

## 访客模块

- 今日访问人数
- 近七天访问趋势
- 浏览总时长
- 重点联系人数量
- 单个访客访问次数
- 最关注内容
- 浏览轨迹及时间节点
- 关注度判断与沟通建议

当前访客列表为演示数据。正式版需要后端事件采集和微信用户授权。

## 我的模块

- 显示脱敏登录手机号
- 显示当前名片状态
- 添加或编辑名片
- 分享名片
- 隐私与数据入口
- 退出登录

## 微信分享说明

网页原型点击“发送给微信联系人”后，会复制名片链接并尝试通过 `weixin://` 唤起微信。

正式微信小程序应使用微信原生分享能力：

- 页面分享配置：`onShareAppMessage`
- 分享按钮：`open-type="share"`
- 分享卡片中携带名片所有者 ID 或短链接参数

出于微信隐私限制，小程序可以打开微信原生分享面板，但不能自动替用户选择某个具体联系人或在未确认的情况下自动发送消息。

## 本地运行

需要 Node.js 18 或更高版本：

```bash
npm install
npm run dev
```

浏览器打开：

```text
http://localhost:5173/
```

生产构建：

```bash
npm run build
```

构建结果位于 `dist/`。

源码保存在 `baige` 仓库的 `ai-card-source` 分支。向该分支推送后，GitHub Actions 会重新构建，并只更新 `gh-pages` 分支中的 `ai-card/` 子目录，不会覆盖原“白鸽慧译”主页。

## 正式上线需要补充

1. 微信手机号授权登录或短信验证码登录。
2. 后端用户、名片、内容和访客数据 API。
3. 图片、视频封面和资料文件的对象存储。
4. 微信 OpenID/UnionID 与内部账号绑定。
5. 名片唯一公开地址、名片 ID 和分享短链接。
6. 页面打开、模块曝光、内容点击、联系方式点击、分享和退出等事件采集。
7. 使用页面可见性和心跳机制计算真实浏览时长。
8. 访客授权、匿名化、数据留存和删除机制。
9. 名片内容审核、敏感信息检测和后台管理权限。
